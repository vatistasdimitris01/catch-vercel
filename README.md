```bash

cd catch-vercel
npm install
npm run dev
```

1. **Tab** or **‚ÜĎ‚Üď** to focus the **vercel token** field
2. Paste your token (get one at [vercel.com/account/tokens](https://vercel.com/account/tokens))
3. **Tab** or **‚Üď** to focus the **deployment id or url** field
4. Paste a deployment ID (`dpl_xxx`) or a full Vercel dashboard URL (`vercel.com/scope/project/id/source`)
5. Hit **Enter** ‚ÄĒ CATCH validates both fields are filled, then starts downloading

If a field is empty when you hit Enter, CATCH shows a red error and moves focus to the empty field.

## How it works

1. **Token** ‚ÄĒ entered directly in the TUI, passed to Vercel API as `Authorization: Bearer <token>`
2. **Deployment** ‚ÄĒ entered directly in the TUI, supports `dpl_xxx` IDs, full Vercel dashboard URLs, or `latest`
3. **Team resolution** ‚ÄĒ auto-detects across all your Vercel teams
4. **File tree** ‚ÄĒ fetched from `vercel.com/api/file-tree/<url>?base=src`
5. **File download** ‚ÄĒ each file's base64 content is fetched from the Vercel files API and decoded
6. **Caching** ‚ÄĒ existing non-empty files are skipped on re-run
